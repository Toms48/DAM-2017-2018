UC-0001 Comprar refresco
Versión 1.0 ( 12/01/2018 )
Autores ?
Fuentes ?
Dependencias Ninguno
Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando el Uusario quiera comprar un refresco.
Precondición PD
Secuencia normal Paso Acción
1 El actor Usuario (ACT-0001) presiona el botón del refresco que quiere para saber el precio.
2 El sistema mostrará el precio del refresco
3 El actor Usuario (ACT-0001) introduce monedas.
4 El sistema analiza la moneda
5 El sistema acumula el dinero introducido.
Postcondición PD
Excepciones Paso Acción
4 Si la moneda no es valida, el sistema devuelve la moneda y vuelve al paso 3, a continuación este caso de uso queda sin efecto
5 Si el importe introducido por el usuario es superior al importe del refresco, el sistema devolverá cambio utilizando los mismos tipos de monedas anteriores, a continuación este caso de uso continúa
5 Si el importe es inferior, el sistema vuelve al paso 3, a continuación este caso de uso continúa
